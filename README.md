# UPnP Port Mapper GUI

Python と `upnpy` を使って、UPnP 対応ルーターのポート開放を GUI で行うツールです。  
`tkinter` ベースなので、コマンドラインに慣れていなくても扱いやすい構成です。

## 特徴

- UPnP 対応ルーターを自動検索
- 複数のルーター / サービス候補から選択可能
- グローバル IP の表示
- 外部ポート / 内部ポート / プロトコル / 内部 IP / 説明を指定してポート開放
- 現在のポートマッピング一覧を取得して表示
- 一覧の内容をフォームへ反映
- 入力値チェック
- ログ表示

## 動作環境

- Python 3.10 以上推奨
- Windows
- 同一 LAN 内にある UPnP 対応ルーター
- ルーター側で UPnP が有効になっていること

## 対応ルーター

このツールは、特定メーカー名で対応を分けているわけではありません。  
コード上は、UPnP IGD のポートマッピング系サービスを持つルーターを対象にしています。

現在の実装で優先的に探しているサービスは次のとおりです。

- `WANIPConnection`
- `WANPPPConnection`
- 上記に近い `wanipconn` / `wanpppconn` を含むサービス ID
- フォールバックとして `connection` を含むサービス

つまり、メーカーではなく「そのルーターがどの UPnP サービスを公開しているか」で対応が決まります。  
実装の判定ロジックは [upnp_gui.py](C:/Users/Yukic/Desktop/Upnp/upnp_gui.py#L286) 付近にあります。

注意:
動作確認済み機種の一覧はまだありません。  
そのため、現時点では「一般的な UPnP IGD 対応ルーター向け」と考えてください。

## インストール

必要な依存関係:

```bash
pip install -r requirements.txt
```

または:

```bash
pip install upnpy
```

`tkinter` は多くの Python 環境で標準搭載されています。

## 起動方法

```bash
python upnp_gui.py
```

## 使い方

1. 起動すると自動で UPnP デバイスの検索が始まります。
2. 候補が複数ある場合は対象のルーターを選択します。
3. `外部ポート`、`内部ポート`、`プロトコル`、`内部 IP`、`説明` を入力します。
4. `ポートを開放 / 更新` を押すとポートマッピングを追加します。
5. `一覧を更新` を押すと現在のマッピング一覧を再取得します。
6. 一覧を選択すると、その内容をフォームへ反映できます。
7. `フォーム内容で削除` を押すと、指定した外部ポートとプロトコルの設定を削除します。

## 注意事項

- UPnP は便利ですが、セキュリティ上のリスクもあります。
- 不要なポートは開放しっぱなしにしないでください。
- ルーター実装によっては、一部の UPnP 機能が使えないことがあります。
- 既に別の端末やアプリが使っているポートは競合で追加できない場合があります。
- セキュリティソフトやルーター設定によっては、検索や追加が失敗することがあります。
